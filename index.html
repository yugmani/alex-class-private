Private Class Fields and Methods in JavaScript Classes

  • The class properties and methods in JavaScript class are by default always public. i.e. they are publicly accessible and modifiable.
  • The private methods can be handy when we want to make class properties and methods inaccessible from the outside of the class they are defined in.

  • Keeping Private

    1. Keeping something private, be it a property or method, it should be accessible only from one place that is the class in which we have defined that property or method.
    2. If someone tries to access private class field/properties or methods from elsewhere JavaScript should not allow it. This includes outside the class in which the class field or method is defined and even any instance of that class.
    3. However, it is possible to access private class field from a method inside the same class.
    4. Syntax: To declare some class field or method as private we have to prefix it with #( the hashtag symbol).

  • Creating class fields and accessing them with methods

    1. Declare a private class field by adding prefix the name of the class field with #. At the time of accessing that private class field, we have to include the #.
    2. Two options to access class fields with methods:
      1. Create new instance of that class and access the property on that instance. And
      2. Declare that property as a static property. In this case, we don't need to instantiate the class to access the property.
    3. Private class fields are designed to be inaccessible from the outside. To overcome from this, create a new method and return the private class field from that method. We can define this method either as public or static.
    4. Same as with static properties, we can call static methods without instantiating the class. If the method is declared as public, we have to instantiate the class. And then we will be able to call that method on your new instance and get the value of private field.

  • Updating private class fields with methods

    1. Same as in creating private class fields, we can modify them as well through method.
    2. To modify and access the private class field, these methods will be available for us to call from the outside.

  • Setters and Getters with Private Class Fields

    1. Now, we are sure that the private class fields are inaccessible from the outside.
    2. Therefore, getter and setter accessors are useless.
    3. If you try to access, or modify private class field from the outside, JavaScript throws error whether there is a setter and/or getter or not.

  • Private Static Class Fields

    1. Accessing private class field with static method is more complicated.
    2. Public class fields and methods are accessible only through class instances. They are not accessible through the classes themselves. So, if you create a static method to access private class field, JavaScript will throw error.
    3. One way to overcome from this problem is to declare the private field as static as well. Now, you can access the static private class field through a static method without instantiating the class.

  • Private Class Fields and Subclasses

    1. We already know that both private class field and method are accessible only from the inside of the class in which they are defined.
    2. This also means that they will be inaccessible for subclasses.
    3. This applies to both public as well as static private class field.

  • Private Methods

    1. Same as private class fields, we can create private methods and same rules of private class fields apply for private methods.
    2. These private methods are accessible only from the inside of the class in which they are defined.
    3. To call private method from outside, we create a new public method and call the private method inside of it.
